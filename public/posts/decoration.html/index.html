3. 配置SSH

其实博主本人并不明确地知道这个步骤是否有用，因为在搭建过程中出了太多奇怪的问题，推倒重建过好几次，但这一步并没有重建。

验证SSH连接是否成功，继续在终端内输入 ssh -T git@github.com
如果返回

1

Hi Username! You've successfully authenticated, but GitHub does not provide shell access.

则为连接成功。如果出现

1

Are you sure you want to continue connecting (yes/no/[fingerprint])?

则输入yes后回车即可。
很不幸，博主在输入yes后返回的是git@github.com: Permission denied (publickey).，搜索发现应该是ssh密钥出了问题，于是返回重新生成.pub文件，不知道为什么反复重试了好几次之后突然成功了（可能靠一些执着感动了它）。
